project

The root object for every project environment defined by means of RapidEnv.

  • name: string, mandatory

    The development environment project name

  • tag: string, optional, default = "main"

    The tag specifies an ID which together with the project name uniquely identifies a development environment on your machine. Since you usually only have more than one development environment concerning the same project on your machine in case you have to work with different branches in parellel this tag will usually specify the branch (e. g. "main"). but it's also possible to work with two different local development environments on the same branch. In this case the tag value has to be specified differently (e. g. "main_a" and main_b").

  • installsourceurl: interpreted string, optional

    The URL where all installation unit packages are taken from by default. If an installunit's "sourceurl" is not defined it automatically defaults to <installsourceurl>/<installunit.space>/<installunit.name>/<installunit.version> whereas each component of "installunit.space" will become a separate subfolder. Otherwise install units define their specific source URL. This property is not mandatory but you are supposed to ether specify a value for it or to specify "sourceurl" for all the project's installunits.

  • installtargetdir: interpreted string, optional

    The relative or absolute path to the local folder where all install units are installed to by default. If an installunit's "homedir" is not defined the "homedir" automatically defaults to <installtargetdir>/<installunit.space>/<installunit.name>/<installunit.version> whereas each component of "space" will become a separate subfolder. Otherwise install units define their specific home directory. This property is not mandatory but you are supposed to ether specify a value for it or to specify a "homedir" for all the project's installunits.

  • profiledir: path to local file, optional, default = "profile"

    The folder where all user profiles are located. By default this value is the folder where the renv command is located.

  • property: child element [0..*]

    The global property definitions.

  • installunit: child element [0..*]

    The install unit definitions

    Set this value to true if you want to use

property

A property is a named value that is either defined centrally (mode = "common") or personally (mode = "personal").

  • name: string, optional

    The property's name.

  • value: interpreted string, optional

    The property's value depending from its mode: common: value simply defines the value. personal: value only defines the default value which interactively can be overwritten.

  • category: single choice of { common | personal }, optional, default = "common"

    Determines if the property's value can and should be individually set by the user or be a common setting for all environments which can be centrally controlled.

    • common: A common property is defined in the environment definition. Its value is not supposed to be changed not be changed personally. Whenever its is changed in the environment definition file all installed environments can overtake this change during the next run of the renv "update" or "configure" command.

    • personal: A personal property can be adapted by a developer.

  • valuetype: single choice of { string | file | path | url }, optional, default = "string"

    Determines the type of this property's value.

    • string: A string of nearly arbitrary length only limited be the OS.

    • file: A (path to a) local file or directory.

    • path: A set of many paths to a local file like the OS command path or the Java class path.

    • url: A path to a remote resource.

  • filetype: single choice of { file | directory }, optional, default = "directory"

    Specifies if a file is a folder / directory or a simple flat file.

    • file: A flat file.

    • directory: A folder.

  • mustExist: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    Only used for properties of type "file". If it is true it is verified if the local file or folder exists.

  • createIfNotExist: boolean (single choice of { 'true' | 'false' }), optional, default = "false"

    Only used for properties of type "file" with file type "directory" and if mustExist = "true". If it is true for properties of type 'common' the path automatically created if it does not exist. For properties of type 'personal' the user is asked for creation if it does not exist.

  • normalize: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    Only used for properties of type "file" and "path". If set to "true" file properties will become normalized which means have canonical path(s). Additionally in "paths" duplicate entries will be removed.

  • interpret: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    If true the specified property value is interpreted. If interpretation is not possible and you do not need it you can turn it off here.

  • interpretdepth: integer, optional, default = "-1"

    If less than 0 the interpretation depth ist unlimited. If greater or equal 1 the interpretation depth is limited.

  • specificvalue: child element [0..*]

    Specific value definition(s) for this property. You might use more than one in case you want to have this property different value for different operating systems.

  • environment: child element [0..*]

    Environment variable definition(s) for this property. You might use more than one in case you want to name the environment variables differently for different operating systems.

  • extension: child element [0..*]

    Extensions extend the property value by being successively appended or prepended.

  • installunitextensions: comma separated list of [0..*]

    Install unit extensions extend the property value by being successively appended or prepended but lead to some automatic migration in case the install unit is upgraded.

specificvalue

A specific value definition for a RapidEnv property This specific value definition overrides the parent property's value attribute for two purposes: 1) platform (os) specific value definitions 2) complex value definitions that can't be performed in an XML attribute and need an XML element

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined the environment variable becomes a platform specific one. If it is undefined the definition the property's value is valid for all platforms.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

  • name: string, mandatory

    The environment variable name

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined the environment variable will become a platform specific one. If it is undefined the definition is valid for all platforms without specific definition.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

extension

A path extension definition for a RapidEnv property containing any path. The platform specific path separator is automatically used in order to prepend or append the extension.

  • propextmode: single choice of { prepend | append | remove }, optional, default = "append"

    Defines weather to "prepend" or "append" this path extension.

    • prepend: Prepend the path extension.

    • append: Append the path extension.

    • remove: Remove an unwanted path extension.

  • value: interpreted string, mandatory

    The environment variable's (default) value. Could be platform specific when using the "osfamily" attribute.

installunit

An installation unit is an installable and configurable unit needed for your development environment e. g the execution platform like JDK, a build tool like Ant, or an IDE like Eclipse.

  • space: string, optional, default = ""

    The installation unit name space. Works like a package in Java.

  • name: string, mandatory

    The installation unit name.

  • version: version, mandatory

    The version that is or should be installed.

  • depends: comma separated list of [0..*]

    Defines the n to n dependency assocition between installation units. Installation of units that depend from other units (depends not null or empty) requires the installation of these units before (this holds for transitive dependencies too). If you install such a unit depending on other ones selectively the required units will be automatically be installed before. If you install all units RapidEnv will choose an appropriate installation sequence. If you deinstall a unit that is required by other ones selectively all other units directly or indirectly requiering this unit will be automatically deinstalled before (after user confirmation).

  • sourcetype: single choice of { url | mavenrepo }, optional, default = "url"

    Defines how to get access to the installation source.

    • url: Installation unit source is defined by a url.

    • mavenrepo: Installation unit source comes from a Maven repository.

  • sourceurl: interpreted string, optional

    Defines where the installation source comes from. If "sourceurl" is not defined it automatically defaults to project.installsourceurl/space/name/version whereas each component of "space" will become a separate subfolder.

  • sourcefile: interpreted string, optional

    Specifies the source file name in case it is not the default one "<install unit name>-<version>.zip". Usually it is sufficient to define either a source file name or a complete source URL. Complex source URLs on the other hand sometimes make it hard or impossible to figure out the source file name. In this case you simply specify the source file name additionally.

  • downloadmode: single choice of { automatic | manual | never }, optional, default = "automatic"

    Describes how to load the installation unit's file(s) (installer).

    • automatic: Automatic download

    • manual: Manual download

    • never: No download at all

  • homedir: interpreted string, optional

    The folder where the unit is installed on your machine. If "homedir" is not defined it automatically defaults to project.installtargetdir/space/name/version whereas each component of "space" will become a separate subfolder.

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined the install unit will exclusievely be installed on this type of operating system otherwise it will be installed operating system independently.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

  • installcontrol: single choice of { normal | optional | discontinued }, optional, default = "normal"

    Determines if to install or deinstall the installation unit. normal: install and update as usual optional: install only when explicitely specified in the installation or update command. Update as usual discontinued: deinstall during update, refuse instsllstion snd configuration

    • normal: Install and update as usual.

    • optional: Install only when explicitely specified in the installation or update command. Update as usual.

    • discontinued: Deinstall during update, refuse installtion and configuration.

  • installmode: single choice of { put | unpack | execute }, optional, default = "unpack"

    Determines how to install the installation unit.

    • put: Simply put a single file into the specified folder.

    • unpack: Unpack a zip, jar, or tar file.

    • execute: Execute an installer.

  • packaging: single choice of { zip | jar | tar | ziptar }, optional

    Determines the type of packaging.

    • zip: zip file.

    • jar: jar file.

    • tar: tar file.

    • ziptar: zipped tar file.

  • unpackstrategy: single choice of { simple | removeroot | removerootdeep }, optional, default = "removerootdeep"

    This property drives the removal of one or more (single) root directories after having unpacked the installation archive file under "homedir()". The default behavior is "removerootdeep" but in most cases "removeroot" will be sufficient and have better performance as well as less problems under Windows file system.

    • simple: Simply put a single file into the specified folder.

    • removeroot: Remove the root folder. Requires the archive to have one single root folder for all files included.

    • removerootdeep: Automatically remove one or more root folders in case there are any. Does not require the archive to have one or more root folders for all files included but in case they are there they are removed.

  • deinstallunusedmode: single choice of { never | always | prompt }, optional, default = "always"

    Determines if or if not to deinstall unused version(s) after upgrade. Per default unused version(s) will always be deinstalled.

    • never: never deinstall unused versions

    • always: always deinstall unused versions

    • prompt: prompt if unused versions should be deinstalled

  • property: child element [0..*]

    Installation unit specific property definitions.

  • download: child element [0..*]

    URLs where to download the installation source

  • sourcefilecheck: child element [0..*]

    Checks on the source file

  • subunit: child element [0..*]

    Subunits if the installation unit. Subunits: - can not be installed specifically before their parent units are installed. - will be automatically installed when their parent units are installed. - will be automatically deinstalled when their parent units are deinstalled (no confirmation). - will be usually installed into the parent units installtion location. f you install all units RapidEnv will choose an appropriate installation sequence.

  • folder: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • icon: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • propertiesfile: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • textfile: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • anttask: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • command: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • xmlfile: child element [0..*]

    Configurations serve to configure an installation unit in distinct phases.

  • installcommand: child element [0..1]

    Only valid for and mandatory for install units of mode "execute"

  • deinstallcommand: child element [0..1]

    Only valid for and mandatory for install units of mode "execute"

  • propextension: child element [0..*]

    Defines a path extension defintion coming from this install unit. The extension will lead to some migration of the property value during upgrade if it changes.

download

Defines a download for one single file.

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined this check will become an operating system specific one. If it is undefined the definition is valid for all platforms without specific definition.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

  • downloadtype: single choice of { automatic | manual }, optional, default = "automatic"

    Describes the method how to download (automatic versus manual))

    • automatic: The file may be loaded automatically by the tool.

    • manual: You have to download the file manually e. g. for license reasons.

  • url: interpreted string, optional

    The URL where you can download the file.

sourcefilecheck

Defines a hash value check on a file.

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined this check will become an operating system specific one. If it is undefined the definition is valid for all platforms without specific definition.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

  • hashalgorithm: single choice of { MD5 | SHA1 }, optional

    The hash algorithm to use for the check.

    • MD5: MD5 hash algorithm

    • SHA1: SHA1 hash algorithm

  • hashvalue: string, optional

    The hash value to check against.

folder

An configuration operation on a single folder

  • installphase: single choice of { preinstall | postinstall | config | predeinstall | postdeinstall }, optional, default = "config"

    The phase when the configuration is executed: - preinstall before first or update installation - postinstall after first or update installation only - config after first or update installation or during explicit configuration - predeinstall before removing the installation - postdeinstall after removing the installation

    • preinstall: before first or update installation

    • postinstall: after first or update installation only

    • config: after first or update installation or during explicit configuration which is automatically done if executing the update command or the config command.

    • predeinstall: before removing the installation

    • postdeinstall: after removing the installation

  • osfamily: single choice of { linux | windows | mac }, optional

    If this value is defined the configuration step gets a platform specific one. If ist is undefined the configuration is valid for all platforms without specific definition.

    • linux: The Linux open source operating system.

    • windows: Microsoft Windows.

    • mac: Apple Mac OS.

  • expressionliteralescaping: boolean (single choice of { 'true' | 'false' }), optional, default = "false"

    Set this value to true if you want to use expression literal escaping

  • cleanupfilesonconfig: interpreted string, optional

    Defines a comma separated list of files that will be deleted after this configuration has been performed

  • path: interpreted string, mandatory

    The path to the file to configure. The path usually is a relative one. If it is relative it is supposed to be relative to the parent installation unit's home directory.

  • sourceurl: interpreted string, optional

    Defines where the file to configure comes from. If "sourceurl" is not defined the file to configure is supposed to be already existent and will only be changed from its original form.

  • deleteafterdeinstall: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    Determines if the configuration file should be deleted after deinstallation in case it is not under the installation root. In the case it is under the installation root it will be deleted anyway. Set this value to false if you want to prevent this default behavior.

  • newline: single choice of { platform | lf | crlf }, optional, default = "platform"

    Defines the newline character(s) to use.

    • platform: The platform specific newline character.

    • lf: Simple newline with one single line feed sign.

    • crlf: Windows specific newline character combination: carriage return + line feed

  • canread: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    Determines if the file to configure can be read (default = true).

  • canwrite: boolean (single choice of { 'true' | 'false' }), optional, default = "true"

    Determines if the file to configure can be written (default = true).

  • canexecute: boolean (single choice of { 'true' | 'false' }), optional, default = "false"

    Determines if the file to configure can be executed (default = false).

  • copycondition: single choice of { notexist | sourcenewer | diff }, optional, default = "sourcenewer"

    The condition when the source is copied over the target: notexist: when the target does not exist sourcenewer: when the source is new than the target: allows manual and automatic change aftwerwards but ensures template changes being processed diff: strict binding to the source file (manual changes are excluded)

    • notexist: Copy only if the target file does not exist. The file can be changed indiviually after installation.

    • sourcenewer: Copy only if the source file is newer that the target file.

    • diff: Copy every time the target file differs from the source file. THis ist the strictest rule.

Usage

renv [-<option> ...] [<command>] [<install unit> ...]

Commands

  • boot, b: boot the environment. This command usually is issued automatically if the environment is not yet set up.

  • config, c: update configuration(s) of the given installation unit(s)

  • deinstall, d: uninstall the given installation unit(s) if installed

  • hashvalue, a: print the hash value of the given file using the given hash algorithm

  • help, h: print command help

  • install, i: install the given installation unit(s) if not installed

  • stat, s: print the installation and configuration status of the given installation unit(s)

  • update, u: update version(s) and configuration(s) of the given installation unit(s)

  • version, v: print RapidEnv version

Options

  • debug, d: debug log level

  • env, e: the folder or file containing the project environment settings file. If it is a folder the settings files is assumed to be 'env.xml'

  • force, f: forces update regardless if something is uptodate or not

  • verbose, v: verbose log level

  • yes, y: batch (yes) run mode without user interaction. If you use this option the "boot", "update", or "configuration" will run without user interaction which causes personal properties to get the default values. Yes / No questions (do you really want to...?) will be automatically answered with "Yes".
